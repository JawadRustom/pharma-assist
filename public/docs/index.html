Introduction

Base URL: http://localhost

This documentation aims to provide all the information you need to work with our API.

As you scroll, you'll see code examples for working with the API in different programming languages in the dark area to the right (or as part of the content on mobile). You can switch the language used with the tabs at the top right (or from the nav menu at the top left on mobile).

Authenticating requests

This API is not authenticated.

Endpoints

POST api/v1/auth/{provider}

Example request:
curl --request POST \
    "http://localhost/api/v1/auth/google|facebook|local" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json"
const url = new URL(
    "http://localhost/api/v1/auth/google|facebook|local"
);

const headers = {
    "Content-Type": "application/json",
    "Accept": "application/json",
};

    method: "POST",

Login

Example request:
curl --request POST \
    "http://localhost/api/v1/login" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --data "{
    \"email\": \"hank.hoeger@example.com\",
    \"password\": \"ipsum\"
}"

const url = new URL(
    "http://localhost/api/v1/login"
);

const headers = {
    "Content-Type": "application/json",
    "Accept": "application/json",
};

let body = {
    "email": "hank.hoeger@example.com",
    "password": "ipsum"
};

fetch(url, {
    method: "POST",
    headers,
    body: JSON.stringify(body),
}).then(response => response.json());

Example response (200, Register Success):


{
    "token": "2|MPvbf6j8OVfoPKuF5bBMUXiE6JrymdQFFHVTHuK1"
}

}

Example response (401, Email or password is wrong or user type not user):


{
    "message": "email or password is wrong"
}

Example response (422, Validation errors):

Request      

POST api/v1/register

Headers

Content-Type      

Example: application/json

Accept      

Example: application/json

Body Parameters

email   string   

Must be a valid email address. Example: pdoyle@example.org

password   string   

Must be at least 8 characters. Must not be greater than 16 characters. Example: ^vIn'5PsM

first_name   string   

Must be at least 2 characters. Must not be greater than 10 characters. Example: ry

last_name   string   

Must be at least 2 characters. Must not be greater than 10 characters. Example: lzd

phone_number   string   

Must be 9 characters. Example: jiadooouy

birth_date   string   

Must be a valid date. Must be a date before 2015-08-09. Example: 2011-03-15

specialty   string   

Must be one of doctor, student, pharmacist, or other. Example: doctor

gender   string   

Must be one of male or female. Example: female
